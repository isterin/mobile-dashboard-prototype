# AGENTS.md - AI Coding Agent Guidelines

## Build & Run Commands

```bash
task dev                                  # Start everything (fe + be + copilot + postgres)

# Individual servers
cd backend && uv run fastapi dev app/main.py   # Backend API (port 8000)
cd frontend && bun run dev                      # Frontend dev server (port 5173)
cd frontend && bun run copilot                  # CopilotKit runtime (port 4000)

# Infrastructure
docker compose -f compose.local.yml up -d       # Start postgres + adminer
docker compose -f compose.local.yml down         # Stop infra
```

### Testing

```bash
cd backend && uv run pytest                                          # All backend tests
cd backend && uv run pytest tests/scripts/test_backend_pre_start.py  # Single file
cd backend && uv run pytest -k "test_init_successful_connection"     # Single test by name
cd frontend && bun run test                    # All Playwright tests (headless)
cd frontend && bun run test:ui                 # Playwright interactive UI mode
```

### Linting & Formatting

```bash
cd backend && uv run ruff check --fix && uv run ruff format   # Lint + format Python
cd backend && uv run mypy app                                  # Type check (strict)
cd frontend && bun run build                   # TypeScript check + Vite build
cd frontend && bun run lint                    # Biome lint + format (auto-fixes)
```

### Database & API Client

```bash
cd backend && uv run alembic upgrade head                          # Run migrations
cd backend && uv run alembic revision --autogenerate -m "desc"     # Create migration
cd backend && uv run alembic downgrade -1                          # Rollback last
bash scripts/generate-client.sh                # Regenerate frontend API client
```

## Architecture

Three servers in dev, unified through Vite proxy (`vite.config.ts`):

| Server             | Port | Purpose                                      |
|--------------------|------|----------------------------------------------|
| Frontend (Vite)    | 5173 | React SPA, proxies `/api` and `/copilotkit`  |
| Backend (FastAPI)  | 8000 | REST API at `/api/v1`, AG-UI agent at `/agent` |
| CopilotKit Runtime | 4000 | Bridges CopilotKit React to AG-UI backend    |

All requests go through the frontend origin — works with ngrok tunnels.

## Project Structure

```
backend/app/
├── api/routes/          # API endpoints (one file per resource)
├── api/deps.py          # Dependency injection (SessionDep)
├── core/                # Config (pydantic-settings), DB engine
├── models.py            # All SQLModel schemas
├── services/            # Business logic
├── agent/               # Google ADK agent (CopilotKit/AG-UI, AWS Bedrock)
└── alembic/versions/    # Database migrations

frontend/src/
├── client/              # Auto-generated API client — DO NOT EDIT
├── components/ui/       # shadcn/ui primitives — DO NOT EDIT
├── components/{Feature}/ # Feature components (Chat/, Common/, MedidataHeader)
├── routes/              # TanStack Router file-based pages
├── hooks/               # Custom React hooks (useMobile, useCustomToast)
└── routeTree.gen.ts     # Auto-generated by TanStack Router — DO NOT EDIT
```

## Critical Rules

1. **Never edit** `frontend/src/client/`, `frontend/src/components/ui/`, or `routeTree.gen.ts`
2. **Always run** `bash scripts/generate-client.sh` after backend model/route changes
3. **Always create Alembic migrations** for database schema changes
4. **Use `SessionDep`** for database access in route handlers

## Code Style — Python (Backend)

Ruff config: Python 3.10 target. Rules: E, W, F, I (isort), B, C4, UP, ARG001, T201.

**Imports** — stdlib → third-party → local (enforced by ruff isort):
```python
import uuid
from typing import Any

from fastapi import APIRouter, HTTPException
from sqlmodel import select

from app.api.deps import SessionDep
```

**Type hints** — Required everywhere (mypy strict). Modern syntax:
```python
def get_item(session: SessionDep, item_id: uuid.UUID) -> Item | None: ...
```

**Models** — SQLModel with Base/Create/Public/Table separation:
```python
class ResourceBase(SQLModel):
    title: str
class ResourceCreate(ResourceBase): pass
class ResourcePublic(ResourceBase):
    id: uuid.UUID
class Resource(ResourceBase, table=True):
    id: uuid.UUID = Field(default_factory=uuid.uuid4, primary_key=True)
```

**Naming**: `snake_case` functions/variables, `PascalCase` classes, `UPPER_SNAKE_CASE` constants.
**Error handling**: `raise HTTPException(status_code=status.HTTP_404_NOT_FOUND, detail="...")`.
**Docstrings**: Required on API endpoints. Google style with `Args:`/`Returns:`.

## Code Style — TypeScript (Frontend)

Biome config: double quotes, no semicolons, 2-space indent, self-closing elements.

**Imports** — external → aliased (`@/`) → relative (enforced by Biome):
```typescript
import { useQuery } from "@tanstack/react-query"

import { cn } from "@/lib/utils"
```

**Components** — Named exports, interface for props, destructured params:
```typescript
interface DashboardProps {
  data: MarketData
  onRefresh: () => void
}

export function Dashboard({ data, onRefresh }: DashboardProps) {
  return (...)
}
```

**Naming**: `PascalCase` components/types/files, `camelCase` hooks/utilities, hooks prefixed `use`.

## API Conventions

- All routes prefixed with `/api/v1`, trailing slashes: `/api/v1/resources/`
- Response models: `{Resource}Public` for single, `{Resource}sPublic` for lists
- List responses include `data` array and `count`
- One router per resource file, with prefix and tags

## Environment

**Root `.env`** (read by backend via `env_file="../.env"`):
- `POSTGRES_*` — Database connection (user/db/password: `marketanalysis`)
- `BACKEND_CORS_ORIGINS` — Comma-separated allowed origins
- `ENVIRONMENT` — `local` | `staging` | `production`

**Frontend `.env`**:
- `VITE_API_URL` — Empty string (relative paths via Vite proxy)
- `VITE_COPILOT_RUNTIME_URL` — `/copilotkit` (proxied to port 4000)
- `VITE_SHOW_DEVTOOLS` — `true`/`false` for React Query/Router devtools
